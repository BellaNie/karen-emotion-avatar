# 需求文档

## 1. 应用概述

### 1.1 应用名称
Karen情绪数字人互动体验网站

### 1.2 应用描述
基于GitHub仓库 https://github.com/BellaNie/karen-emotion-avatar 的 emotion-avatar-demo 子项目构建的在线互动网站。通过浏览器摄像头识别用户面部情绪，实时播放对应的数字人动画，提供完整的情绪互动体验与统计功能。技术栈为 Vite + Vanilla JavaScript，所有处理在浏览器本地完成，无需后端服务。

## 2. 用户与使用场景

### 2.1 目标用户
- 对情绪识别技术感兴趣的体验者
- 希望了解自身情绪状态的用户
- 数字人互动技术的探索者

### 2.2 核心使用场景
- 用户通过摄像头与数字人Karen进行情绪互动
- 查看当日情绪识别统计与分布
- 无摄像头环境下通过键盘演示各情绪动画
- 开发者通过调试模式查看识别详情

## 3. 页面结构与功能说明

### 3.1 页面结构
```
主页面
├── 摄像头面板（左侧）
├── 数字人头像舞台（居中）
├── 情绪记录浮动标签（右侧）
├── 情绪统计日报（可展开）
├── 办公室吐槽气泡（浮动）
├── 背景装饰元素
└── 调试面板（debug模式）
```

### 3.2 功能说明

#### 3.2.1 摄像头人脸情绪识别
- 使用 face-api.js 库（Tiny Face Detector + 表情识别模型）在浏览器本地识别用户表情
- 识别情绪类型：neutral（平静）、happy（开心）、sad（低落）、angry（烦躁）、fearful（焦虑）、disgusted（倦怠）、surprised（惊讶）
- 摄像头数据不上传，所有处理在本地完成
- face-api.js 库文件通过 jsDelivr CDN 加载
- face-api.js 模型文件通过 jsDelivr CDN 加载（指向该 GitHub 仓库）

#### 3.2.2 数字人动画播放
- 根据识别到的情绪播放对应的 MP4 视频动画
- 视频文件通过 jsDelivr CDN 从该 GitHub 仓库加载（路径：emotion-avatar-demo/public/avatar/）
- 支持7种情绪对应的动画视频
- 动画在大型圆形头像舞台居中播放

#### 3.2.3 状态机流程
- BOOT：启动状态
- IDLE：空闲状态
- OBSERVING：观察识别状态
- EMOTION_LOCKED：情绪锁定状态
- FEEDBACK：反馈播放状态
- RESET：重置状态
- 完整的互动流程循环

#### 3.2.4 情绪统计日报
- 记录当日情绪识别次数
- 展示情绪分布条形图
- 显示最近情绪识别记录
- 支持展开/收起

#### 3.2.5 键盘演示模式
- 无摄像头时可按数字键 1-7 手动触发各情绪动画
- 1-7 分别对应7种情绪类型

#### 3.2.6 调试模式
- URL 添加 ?debug=true 参数启用
- 显示调试面板
- 展示人脸检测框
- 显示情绪识别置信度
- 显示当前状态机状态

#### 3.2.7 UI设计
- 暗色博物馆风格，黑色背景
- 金色/暖白文字
- glassmorphism 浮动卡片效果
- 大型圆形头像舞台居中
- 左侧摄像头面板
- 右侧情绪记录浮动标签
- 浮动的办公室吐槽气泡（内容包括「下周要上线，代码还没写」等）
- 大型背景文字「KAREN」
- 旋转装饰圆盘
- 响应式布局，支持桌面和移动端
- 情绪主题色调：
  - warm（开心）
  - cool（低落）
  - hot（烦躁）
  - alert（焦虑）
  - muted（倦怠）
  - bright（惊讶）
  - stable（平稳）

## 4. 业务规则与逻辑

### 4.1 情绪识别流程
1. 用户进入网站，系统请求摄像头权限
2. 摄像头启动后，系统进入OBSERVING状态
3. 持续检测人脸并识别情绪
4. 识别到明确情绪后进入EMOTION_LOCKED状态
5. 播放对应情绪的数字人动画（FEEDBACK状态）
6. 动画播放完成后进入RESET状态
7. 返回IDLE状态，等待下一次识别

### 4.2 情绪统计规则
- 每次成功识别情绪后记录到当日统计
- 统计数据包含：情绪类型、识别时间、识别次数
- 条形图按情绪类型分组展示识别次数
- 最近情绪记录按时间倒序排列

### 4.3 资源加载规则
- face-api.js 库文件通过 jsDelivr CDN 加载
- face-api.js 模型文件通过 jsDelivr CDN 加载（指向 GitHub 仓库 https://github.com/BellaNie/karen-emotion-avatar）
- 数字人视频文件通过 jsDelivr CDN 从该 GitHub 仓库加载（路径：emotion-avatar-demo/public/avatar/）
- 所有资源加载失败时显示错误提示

### 4.4 键盘演示模式规则
- 检测到无摄像头权限或摄像头不可用时自动启用
- 按数字键 1-7 分别触发对应情绪动画
- 演示模式下不进行实际情绪识别
- 演示触发的情绪不计入统计

### 4.5 调试模式规则
- URL 包含 ?debug=true 参数时启用
- 显示人脸检测框覆盖在摄像头画面上
- 实时显示各情绪的识别置信度数值
- 显示当前状态机所处状态
- 调试信息不影响正常功能使用

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户拒绝摄像头权限 | 显示提示信息，自动启用键盘演示模式 |
| 摄像头不可用 | 显示错误提示，自动启用键盘演示模式 |
| 未检测到人脸 | 保持OBSERVING状态，持续检测 |
| 情绪识别置信度过低 | 不触发动画，继续观察 |
| 视频文件加载失败 | 显示加载错误提示，跳过该情绪动画 |
| 模型文件加载失败 | 显示错误提示，无法启动识别功能 |
| 多张人脸同时出现 | 识别置信度最高的人脸 |
| 网络连接中断 | 已加载资源继续使用，未加载资源显示错误 |

## 6. 验收标准

1. 用户访问网站，授权摄像头权限
2. 摄像头启动，用户面部出现在左侧面板
3. 用户做出明显表情（如微笑）
4. 系统识别到情绪，中央数字人播放对应动画
5. 动画播放完成，右侧显示情绪记录标签
6. 用户点击展开情绪统计日报，查看当日情绪分布条形图
7. 用户在URL添加 ?debug=true，刷新页面，调试面板显示人脸检测框和置信度数据
8. 用户拒绝摄像头权限，按数字键 3，数字人播放sad情绪动画
