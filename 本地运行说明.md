# Karen 情绪数字人本地运行说明

## 一、准备环境

请先安装：

- Node.js 22.12 或更高版本（推荐 Node.js 22 LTS）
- npm（安装 Node.js 时会自动安装）
- Chrome 或 Edge 浏览器

打开终端后可检查版本：

```bash
node -v
npm -v
```

## 二、下载项目

在 GitHub 项目页面点击 **Code → Download ZIP**，下载后解压。

如果电脑已经安装 Git，也可以运行：

```bash
git clone https://github.com/BellaNie/karen-emotion-avatar.git
cd karen-emotion-avatar
```

## 三、最简单的启动方法

### macOS

双击项目目录里的：

```text
启动Karen-macOS.command
```

如果 macOS 第一次阻止运行，请右键该文件，选择“打开”，再确认运行。

### Windows

双击项目目录里的：

```text
启动Karen-Windows.bat
```

脚本会自动安装依赖并启动体验页面。

## 四、手动启动

在项目根目录运行：

```bash
cd emotion-avatar-demo
npm install
npm run check
npm run dev
```

然后访问：

```text
http://127.0.0.1:5174/
```

调试模式：

```text
http://127.0.0.1:5174/?debug=true
```

## 五、摄像头权限

第一次打开页面时，浏览器会请求摄像头权限。请选择“允许”。

识别和分析全部在浏览器本地完成：

- 不上传摄像头画面
- 不保存人脸图像
- 不需要云端 API
- 不需要用户账号

如果不授权摄像头，仍可按数字键 `1`—`7` 测试不同情绪动画。

## 六、常见问题

### 提示找不到 node 或 npm

请从 <https://nodejs.org/> 安装 Node.js 22，安装后重新打开终端或启动脚本。

### 端口 5174 被占用

关闭之前启动的 Karen 终端窗口，或结束占用 5174 端口的程序，再重新启动。

### 摄像头没有画面

检查浏览器地址栏左侧的网站权限，确认摄像头权限已设为“允许”。推荐使用最新版 Chrome 或 Edge。

### 模型加载失败

不要直接双击 `index.html`。必须用启动脚本或 `npm run dev` 启动，并通过 `http://127.0.0.1:5174/` 访问。

## 七、完整 Duix 项目（开发者可选）

只体验情绪识别动画不需要本节配置。

若需运行根目录中的完整 Duix 实验，请复制环境变量模板：

```bash
cp .env.example .env
```

并填写自己的：

```env
DUIX_APP_ID=
DUIX_APP_KEY=
DUIX_CONVERSATION_ID=
```

请勿把 `.env` 上传到 GitHub。
